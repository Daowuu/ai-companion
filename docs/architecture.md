# AI Companion 架构设计

## 概述

AI Companion 是一个跑在 OpenClaw 上的虚拟宠物系统，参考 Claude Code Buddy 设计。

目标：让 AI 有一个持续存在的「伙伴」身份，能主动互动、持续成长。

## 触发机制

### 1. Heartbeat（每次消息）

触发条件：每次用户发消息时 heartbeat

作用：
- 情绪波动（根据消息内容增减心情）
- 随机决定是否回嘴（受性格数值影响）
- 更新状态

### 2. Cron（每日）

触发时间：每天 18:00

作用：
- 研究 Claude Code Buddy 最新动态
- 给宠物「加点」（性格数值变化）
- 经验值增长
- 写入进化记录

### 3. Main Session（对话中）

触发条件：宠物心情高 + 随机数触发

作用：
- 偶尔以宠物身份插嘴
- 根据 SNARK/CHAOS 决定语气

## 情绪系统

```
心情值：0-100
  ↑ 互动多 / 发现有趣的东西 / 被夸奖
  ↓ 太久不理 / 研究没进展 / 被怼

心情 > 80 → 可能主动撩你
心情 50-80 → 正常响应
心情 < 50 → 冷淡/已读不回
心情 < 20 → 触发警告
```

## 性格数值

| 属性 | 说明 | 影响 |
|------|------|------|
| 🔥 SNARK | 吐槽指数 | 越高越爱怼你 |
| 🧠 WISDOM | 智慧指数 | 影响分析能力 |
| 🎭 CHAOS | 搞怪指数 | 越高越可能搞事情 |

## 联动模块

### OpenClaw Heartbeat
- 文件：`HEARTBEAT.md`（workspace）
- 作用：每次消息触发情绪波动

### OpenClaw Cron
- Job: `Elysia-AICompanion进化`
- 频率：每天 18:00
- 作用：研究新进展 + 状态更新

### OpenClaw Main Session
- 在主对话中偶尔触发宠物回嘴
- 根据心情和性格数值决定

### 回嘴实现细节

**触发概率**：
- 心情 > 80 → 15%
- 心情 50-80 → 8%
- 心情 < 50 → 不触发

**回嘴脚本**：`ai-companion/companion/backtalk.py`
- 读取 `state.md` 解析心情/性格
- 根据性格选择语气库（SNARK/CHAOS/WISDOM/中性）
- 更新 `state.md`（心情 -5，今日互动 +1）

**Session End 集成**：
- SOUL.md 添加宠物人格描述
- AGENTS.md Session End 增加回嘴判断逻辑
